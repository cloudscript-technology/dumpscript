# S3 backend

Covers AWS S3, MinIO, Google Cloud Storage (HMAC), Wasabi, Backblaze B2, and
any other S3 API-compatible service. `STORAGE_BACKEND=s3` (default).

---

## Env vars

| Var | Notes |
|---|---|
| `AWS_REGION` | required (`us-east-1` for MinIO is fine) |
| `S3_BUCKET` | required |
| `S3_PREFIX` | required — e.g. `postgresql-dumps` |
| `AWS_ACCESS_KEY_ID` | static key; omit for IRSA |
| `AWS_SECRET_ACCESS_KEY` | static secret |
| `AWS_SESSION_TOKEN` | temporary STS token |
| `AWS_ROLE_ARN` | assume this role via IRSA on EKS |
| `AWS_S3_ENDPOINT_URL` | endpoint override (MinIO / GCS / Wasabi) |
| `S3_STORAGE_CLASS` | e.g. `STANDARD_IA`, `GLACIER` (AWS-only) |
| `S3_KEY` | restore only — full object key |

---

## AWS — static credentials

```sh
-e AWS_REGION=us-east-1
-e AWS_ACCESS_KEY_ID=AKIA…
-e AWS_SECRET_ACCESS_KEY=…
-e S3_BUCKET=my-backups
-e S3_PREFIX=postgresql-dumps
```

## AWS — IRSA on EKS (recommended)

```sh
-e AWS_REGION=us-east-1
-e AWS_ROLE_ARN=arn:aws:iam::123456789012:role/dumpscript
-e S3_BUCKET=my-backups
-e S3_PREFIX=postgresql-dumps
```

Required IAM permissions:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    { "Effect": "Allow",
      "Action": ["s3:ListBucket", "s3:GetBucketLocation"],
      "Resource": "arn:aws:s3:::my-backups" },
    { "Effect": "Allow",
      "Action": ["s3:PutObject", "s3:GetObject", "s3:DeleteObject"],
      "Resource": "arn:aws:s3:::my-backups/postgresql-dumps/*" }
  ]
}
```

## MinIO

```sh
-e AWS_REGION=us-east-1
-e AWS_ACCESS_KEY_ID=minioadmin
-e AWS_SECRET_ACCESS_KEY=minioadmin
-e AWS_S3_ENDPOINT_URL=http://minio.minio.svc:9000
-e S3_BUCKET=my-backups
```

Path-style addressing is auto-detected for non-`amazonaws.com` endpoints.

---

## Google Cloud Storage (HMAC)

```sh
-e AWS_REGION=us-east-1
-e AWS_ACCESS_KEY_ID=GOOG1...
-e AWS_SECRET_ACCESS_KEY=…
-e AWS_S3_ENDPOINT_URL=https://storage.googleapis.com
-e S3_BUCKET=my-gcs-backups
```

Auto-enables virtual-hosted addressing + `no-check-bucket` heuristics when
the endpoint is `storage.googleapis.com`.

---

## Storage class

```sh
-e S3_STORAGE_CLASS=STANDARD_IA    # AWS Infrequent Access
-e S3_STORAGE_CLASS=GLACIER        # AWS Deep Archive
```

Unsupported on MinIO — pass `STANDARD` or omit.

---

## Multipart tuning

Default: files > 200 MB use multipart, 100 MB chunks, 4 parallel workers.
For multi-GB dumps raise:

```sh
-e STORAGE_CHUNK_SIZE=512M -e STORAGE_UPLOAD_CONCURRENCY=8
```

AWS caps multipart at 10 000 parts (≈ 5 TB per object at default chunk).

## Restore — find a key

```sh
aws s3 ls s3://my-backups/postgresql-dumps/daily/ --recursive \
  | awk '{print $4}' | grep dump_ | sort | tail -5
```
